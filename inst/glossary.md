# mysterycall Glossary

Domain-specific terms used throughout the mysterycall package and mystery caller
study methodology.

---

## Study Design Terms

**audit study**
A research method in which trained callers pose as patients to measure access to
healthcare services without revealing the study purpose to the provider's office.
Synonymous with *mystery caller study*.

**mystery caller**
A trained research assistant who phones a provider's office posing as a prospective
patient to assess appointment availability, insurance acceptance, and wait times.

**lab assistant**
Internal term for a mystery caller; used in function arguments (e.g.,
`lab_assistant_names`) to avoid confusion with clinical terms.

**Phase 1 data**
The pre-call roster: a data frame or spreadsheet containing provider names, NPIs,
phone numbers, practice names, and state information prepared before calls begin.

**Phase 2 data**
The post-call data collected by lab assistants during or after telephone encounters,
including appointment availability, hold times, transfer counts, and exclusion
reasons.

**for_redcap**
A flag column in Phase 1 data indicating whether a provider row should be uploaded
to the REDCap data entry system (`"Yes"` / `"No"`).

---

## Provider Identity Terms

**NPI** (National Provider Identifier)
A unique 10-digit identifier assigned to every US healthcare provider by CMS.
All NPIs satisfy the Luhn checksum algorithm.

**NPI validation**
The process of verifying that an NPI is exactly 10 digits and passes the Luhn
checksum. Performed by `mysterycall_validate_npi()`.

**taxonomy**
A NUCC (National Uniform Claim Committee) code and description that classifies
a provider's specialty. The `mysterycall::taxonomy` dataset contains all valid
taxonomy codes. Used by `mysterycall_search_taxonomy()` to query the NPI registry.

**taxonomy description**
The human-readable string for a NUCC taxonomy code (e.g., `"Gynecologic Oncology"`).
Must match the NPI registry exactly for searches to return results.

**subspecialty**
The clinical subspecialty of an OB/GYN physician in the `mysterycall::physicians`
dataset. One of seven values (Gynecologic Oncology, Maternal-Fetal Medicine, etc.).

**ACOG District**
One of 11 geographic regions defined by the American College of Obstetricians and
Gynecologists. Used for stratified sampling and reporting. Stored in
`mysterycall::ACOG_Districts`.

## Data Pipeline Terms

**audit trail**
A JSON file written alongside every Phase 1 output CSV by `mysterycall_clean_phase1()`
documenting row counts, timestamps, and processing flags for each run.

**processing flag**
A boolean column (e.g., `processing_flag_empty_name`) appended to output data by
`mysterycall_clean_phase1()` to document why a row was modified.

**quality tier**
A categorical rating (`"Excellent"`, `"Good"`, `"Fair"`, `"Poor"`) computed by
`mysterycall_quality_tier()` from a 0–1 quality score, used to summarise data
quality in workflow summaries.

**retention rate**
The fraction of input rows that remain in the output after cleaning or filtering.
Reported in `workflow_summary` by `mysterycall_run_workflow()`.

**not-contacted state**
A US state in which no providers were successfully reached during the study period.
Reported by `mysterycall_not_contacted_states()`.

**workload split**
The process of dividing the Phase 1 call list evenly among lab assistants by
`mysterycall_split_and_save()`. Each assistant receives a separate Excel workbook.

---

## Geospatial Terms

**isochrone**
A polygon representing the area reachable within a given drive time from a provider
location. Generated by `mysterycall_create_isochrones()` via the HERE Routing API.

**drive-time band**
One isochrone polygon at a specific travel time threshold (e.g., 30 min, 60 min,
120 min). Multiple bands per provider are stacked for coverage analysis.

**block group**
A US Census geographic unit smaller than a census tract. Used by
`mysterycall_map_block_group()` to assess how many residents live within each
isochrone catchment area.

**HRR** (Hospital Referral Region)
A geographic area defined by the Dartmouth Atlas based on patterns of hospital use.
Loaded by `mysterycall_hrr()` from the Dartmouth Atlas boundary file.

**overlap fraction**
The proportion of a block group's area that falls inside an isochrone polygon.
Computed by `mysterycall_calculate_overlap()`.

**geocode**
The process of converting a text address to geographic coordinates (latitude,
longitude). Performed by `mysterycall_geocode()` via the Google Maps Geocoding API.

---

## API and Infrastructure Terms

**NPI registry**
The public NPPES (National Plan and Provider Enumeration System) database maintained
by CMS, accessible via the `npi` package (`npi::npi_search()`).

**snapshot**
An `.rds` file written by `mysterycall_search_taxonomy()` capturing the raw API
response. Enables reproducible re-runs without re-querying the live registry.

**cache**
In-memory memoisation of isochrone API responses within a single R session via
the `memoise` package. Cleared by `mysterycall_clear_isochrone_cache()`.

**write_snapshot**
A parameter (`logical`, default `TRUE`) controlling whether `mysterycall_search_taxonomy()`
saves an `.rds` snapshot to `output_dir`. Set to `FALSE` in unit tests.

**notify**
A parameter (`logical`) controlling whether `beepr::beep()` sounds are played on
function completion. Requires the `beepr` package in Suggests. Set to `FALSE`
in all automated tests and CI.
